title: 'Service'
description: 'Base class for all generated service implementations'
---

The `Service` class is the base class for all service implementations generated by the `@Service` macro. It provides the foundational properties and initialization logic for making HTTP requests.

## Declaration

```swift
open class Service
```

## Properties

<ResponseField name="baseURL" type="URL" required>
  The base URL for all requests made by this service. All endpoint paths are appended to this base URL.
</ResponseField>

<ResponseField name="executor" type="ServiceExecutor" required>
  The executor responsible for executing HTTP requests. Defaults to `URLSessionServiceExecutor`.
</ResponseField>

## Initializers

### Primary Initializer

```swift
public init(baseURL: URL, executor: ServiceExecutor = URLSessionServiceExecutor())
```

<ParamField path="baseURL" type="URL" required>
  The base URL for the service.
</ParamField>

<ParamField path="executor" type="ServiceExecutor" default="URLSessionServiceExecutor()">
  The service executor to use for making requests.
</ParamField>

**Usage:**

```swift
let baseURL = URL(string: "https://api.example.com")!
let service = UserServiceImpl(
    baseURL: baseURL,
    executor: URLSessionServiceExecutor()
)
```

### String-based Initializer

```swift
convenience public init(baseURL: String, executor: ServiceExecutor = URLSessionServiceExecutor())
```

<ParamField path="baseURL" type="String" required>
  The base URL string for the service. Will cause a fatal error if the string is not a valid URL.
</ParamField>

<ParamField path="executor" type="ServiceExecutor" default="URLSessionServiceExecutor()">
  The service executor to use for making requests.
</ParamField>

**Usage:**

```swift
let service = UserServiceImpl(
    baseURL: "https://api.example.com",
    executor: URLSessionServiceExecutor()
)
```

### URLSession Initializer (URL)

```swift
convenience public init(baseURL: URL, session: URLSession)
```

<ParamField path="baseURL" type="URL" required>
  The base URL for the service.
</ParamField>

<ParamField path="session" type="URLSession" required>
  The URLSession to use for making requests. Automatically wraps it in a `URLSessionServiceExecutor`.
</ParamField>

**Usage:**

```swift
let configuration = URLSessionConfiguration.default
configuration.timeoutIntervalForRequest = 30
let session = URLSession(configuration: configuration)

let service = UserServiceImpl(
    baseURL: URL(string: "https://api.example.com")!,
    session: session
)
```

## Usage Examples

### Basic Service Creation

```swift
@Service
protocol UserService {
    @GET("users")
    func getUsers() async throws -> [User]
}

// Create service instance
let service = UserServiceImpl(baseURL: "https://api.example.com")
let users = try await service.getUsers()
```

### Custom Executor

```swift
class MyCustomExecutor: ServiceExecutor {
    func execute(_ request: Request) async throws -> (Data, HTTPURLResponse) {
        // Custom implementation
    }
}

let service = UserServiceImpl(
    baseURL: "https://api.example.com",
    executor: MyCustomExecutor()
)
```

### Custom URLSession Configuration

```swift
let configuration = URLSessionConfiguration.default
configuration.timeoutIntervalForRequest = 60
configuration.httpAdditionalHeaders = ["User-Agent": "MyApp/1.0"]

let session = URLSession(configuration: configuration)
let service = UserServiceImpl(
    baseURL: "https://api.example.com",
    session: session
)
```

## Related: ServiceContainer

SwiftRequest also provides a `ServiceContainer` class as an alternative base for service implementations:

```swift
public class ServiceContainer {
    let baseURL: URL
    let session: URLSession
    
    public init(baseURL: URL, session: URLSession = .shared)
    convenience public init(baseURL: String, session: URLSession = .shared)
}
```

This class is similar to `Service` but holds a `URLSession` directly instead of a `ServiceExecutor`. It's primarily used for simpler use cases or backward compatibility.

**Properties:**
- `baseURL`: The base URL for requests
- `session`: The URLSession for making requests

<Note>
The `Service` class with `ServiceExecutor` is the recommended approach as it provides more flexibility for custom networking implementations.
</Note>

## Notes

- The `Service` class is `open`, allowing you to subclass it if needed
- All generated service implementations (created by `@Service` macro) inherit from this class
- The string-based initializers will cause a fatal error if the provided string is not a valid URL
- When using custom `URLSession`, it's automatically wrapped in a `URLSessionServiceExecutor`
