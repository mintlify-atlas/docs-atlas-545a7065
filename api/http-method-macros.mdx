---
title: 'HTTP Method Macros'
description: 'Macros for defining HTTP request methods on service protocol functions'
---

HTTP method macros are attached peer macros that mark protocol methods as HTTP endpoints. Each macro corresponds to a standard HTTP method and generates the implementation that constructs and executes the request.

## Available Macros

### @GET

```swift
@attached(peer)
public macro GET(_ path: String = "")
```

<ParamField path="path" type="String" default="">
  The endpoint path for the GET request. Can include path parameters using `{paramName}` syntax.
</ParamField>

**Usage:**

```swift
@GET("users")
func getUsers() async throws -> [User]

@GET("users/{id}")
func getUser(@Path id: String) async throws -> User
```

### @POST

```swift
@attached(peer)
public macro POST(_ path: String = "")
```

<ParamField path="path" type="String" default="">
  The endpoint path for the POST request. Can include path parameters using `{paramName}` syntax.
</ParamField>

**Usage:**

```swift
@POST("users")
func createUser(@Body user: User) async throws -> User

@POST("login")
func login(@FormField username: String, @FormField password: String) async throws -> Token
```

## Path Parameters

All HTTP method macros support path parameters using the `{paramName}` syntax. These are replaced at runtime with values from parameters annotated with `@Path`:

```swift
@GET("users/{userId}/posts/{postId}")
func getPost(@Path userId: String, @Path postId: String) async throws -> Post
```

## Notes

- All methods must be `async throws` functions
- The path parameter is optional and defaults to an empty string
- Path parameters must correspond to function parameters marked with `@Path`
- Return types can be any `Decodable` type, `Data`, or `Void`
- Combine with parameter macros (`@Path`, `@QueryParam`, `@Header`, `@Body`, `@FormField`) to build complete requests
