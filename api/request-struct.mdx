The `Request` struct is a data structure that encapsulates all the information needed to make an HTTP request. It is created by the generated service methods and passed to the `ServiceExecutor` for execution.

## Declaration

```swift
public struct Request
```

## Properties

<ResponseField name="url" type="URL" required>
  The complete URL for the request, including the base URL and path.
</ResponseField>

<ResponseField name="method" type="String" required>
  The HTTP method (e.g., "GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS", "HEAD").
</ResponseField>

<ResponseField name="queryParams" type="Params?">
  Dictionary of query parameters to append to the URL. Keys are parameter names, values are optional `CustomStringConvertible` values.
</ResponseField>

<ResponseField name="headers" type="Params?">
  Dictionary of HTTP headers. Keys are header names, values are optional `CustomStringConvertible` values.
</ResponseField>

<ResponseField name="formFields" type="Params?">
  Dictionary of form fields for URL-encoded form data. Keys are field names, values are optional `CustomStringConvertible` values.
</ResponseField>

<ResponseField name="body" type="(any Encodable)?">
  The request body to be JSON-encoded. Must conform to `Encodable`.
</ResponseField>

## Type Aliases

### Params

```swift
public typealias Params = [String: (any CustomStringConvertible)?]
```

A dictionary type for parameters where:
- **Key**: Parameter name (`String`)
- **Value**: Optional value conforming to `CustomStringConvertible`

Used for query parameters, headers, and form fields.

## Usage Examples

### Simple GET Request

```swift
let request = Request(
    url: URL(string: "https://api.example.com/users")!,
    method: "GET"
)
```

### GET Request with Query Parameters

```swift
let request = Request(
    url: URL(string: "https://api.example.com/users")!,
    method: "GET",
    queryParams: [
        "page": 1,
        "limit": 10,
        "sort": "name"
    ]
)
```

### POST Request with JSON Body

```swift
struct User: Codable {
    let name: String
    let email: String
}

let newUser = User(name: "John Doe", email: "john@example.com")

let request = Request(
    url: URL(string: "https://api.example.com/users")!,
    method: "POST",
    headers: ["Authorization": "Bearer token123"],
    body: newUser
)
```

### POST Request with Form Fields

```swift
let request = Request(
    url: URL(string: "https://api.example.com/login")!,
    method: "POST",
    formFields: [
        "username": "john",
        "password": "secret"
    ]
)
```

### Request with Headers and Path Parameters

```swift
let request = Request(
    url: URL(string: "https://api.example.com/users/123")!,
    method: "GET",
    headers: [
        "Authorization": "Bearer token123",
        "Accept": "application/json"
    ]
)
```

## Extension Methods

### asURLRequest()

```swift
public extension Request {
    func asURLRequest() throws -> URLRequest
}
```

Converts a `Request` to a standard `URLRequest` for use with `URLSession` or other networking libraries.

This method:
- Applies query parameters to the URL
- Sets the HTTP method
- Encodes form fields as URL-encoded data with `Content-Type: application/x-www-form-urlencoded`
- Encodes the body as JSON with `JSONEncoder` (or uses raw `Data` if body is `Data`)
- Adds all headers to the request

**Throws**: An error if encoding fails.

**Example**:

```swift
let request = Request(
    url: URL(string: "https://api.example.com/users")!,
    method: "POST",
    headers: ["Authorization": "Bearer token"],
    body: newUser
)

let urlRequest = try request.asURLRequest()
// Use with URLSession or other libraries
```

This extension is defined in Sources/SwiftRequest/Request+toURLRequest.swift and is used internally by executors to convert SwiftRequest's `Request` type to Foundation's `URLRequest`.

## Notes

- In typical usage, you don't create `Request` instances manually; they are generated by the `@Service` macro
- All parameter dictionaries use optional values to allow `nil` values that can be omitted
- The `Params` type requires values to conform to `CustomStringConvertible` for easy conversion to strings
- `body` and `formFields` are mutually exclusive in practice (don't use both in the same request)
- Query parameters and form fields are automatically URL-encoded
- The `Request` struct is converted to a `URLRequest` by the `ServiceExecutor` before execution using `asURLRequest()`
